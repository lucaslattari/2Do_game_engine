    Estrutura do Projeto 2Do

    Este documento descreve a estrutura de diretórios e arquivos do projeto 2Do, explicando a função de cada componente e como eles se relacionam para criar o motor de jogos 2D.

    Visão Geral da Estrutura

    Descrição dos Arquivos e Diretórios

    Arquivos Principais

    • main.py: Ponto de entrada do jogo. Responsável por inicializar o Pygame, carregar o jogo e executar o loop principal.
    • game.py: Contém a classe Game, que gerencia o estado do jogo, carrega níveis, e coordena a renderização e atualização de entidades.
    • config.ini: Arquivo de configuração do jogo, onde são definidos parâmetros como resolução, volume e plano de fundo.

    Módulos do Motor

    • asset_manager.py: Gerencia o carregamento e organização de assets (recursos) como sprites, tiles e mapas.
    • background.py: Controla o plano de fundo do jogo, incluindo rolagem parallax e carregamento de imagens.
    • entity.py: Define as classes base para entidades do jogo, como Tile e Entity, que são usadas para representar objetos no jogo.
    • input_handler.py: Gerencia a entrada do usuário (teclado) e mapeia teclas para ações no jogo.
    • player.py: Implementa a classe Player, que representa o personagem controlado pelo jogador, incluindo movimentação, animações e colisões.
    • platformer.py: Define a classe Platform, que representa as plataformas do jogo.
    • item.py: Define a classe Item, que representa itens coletáveis no jogo.
    • utils.py: Contém funções utilitárias, como renderização de FPS e leitura de arquivos de configuração.

    Documentação

    • docs/: Diretório contendo a documentação do projeto em formato Markdown.
    • assets.md: Explica como gerenciar assets (sprites, tiles, etc.).
    • configuracao.md: Detalha o arquivo config.ini e como configurar o jogo.
    • controle.md: Explica como configurar e personalizar os controles do jogo.
    • entidades.md: Descreve as entidades do jogo (player, itens, plataformas).
    • estrutura_projeto.md: Este arquivo, que descreve a estrutura do projeto.
    • executando.md: Guia para executar o jogo e configurar o ambiente.
    • logica_jogo.md: Explica a lógica principal do jogo, como física e colisões.
    • plano_fundo.md: Detalha como o plano de fundo é configurado e renderizado.
    • referencias.md: Lista referências e recursos utilizados no projeto.
    • utils.md: Explica as funções utilitárias do projeto.

    Outros Arquivos

    • requirements.txt: Lista as dependências do projeto, como Pygame e PyTMX.
    • mkdocs.yml: Arquivo de configuração para gerar a documentação usando MkDocs.
    • site/: Diretório gerado pelo MkDocs contendo a documentação em formato HTML.

    Fluxo de Execução

    1. Inicialização: O jogo começa no main.py, que inicializa o Pygame e carrega o arquivo de configuração (config.ini).
    2. Carregamento do Nível: O game.py carrega o nível usando o AssetManager e inicializa as entidades (player, itens, plataformas).
    3. Loop Principal:
    4. O InputHandler processa a entrada do usuário.
    5. O Game atualiza o estado do jogo (física, colisões, animações).
    6. O Background atualiza e renderiza o plano de fundo.
    7. Todas as entidades são renderizadas na tela.
    8. Finalização: Quando o jogo é encerrado, o Pygame é finalizado e os recursos são liberados.

    Relacionamento entre Módulos

    • O main.py depende do game.py para gerenciar o estado do jogo.
    • O game.py depende do asset_manager.py para carregar recursos e do input_handler.py para processar entradas.
    • As entidades (player.py, item.py, platformer.py) dependem da classe base entity.py.
    • O background.py é usado pelo game.py para renderizar o plano de fundo.
    • Funções utilitárias no utils.py são usadas em vários módulos.
