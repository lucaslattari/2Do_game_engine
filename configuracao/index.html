    Configuração e Setup do 2Do

    Este guia irá ajudá-lo a configurar o ambiente de desenvolvimento para o 2Do e entender as configurações básicas do framework.

    Preparando o Ambiente

    Requisitos de Sistema Detalhados

    Para garantir o funcionamento adequado do 2Do, seu sistema deve atender aos seguintes requisitos:

    • Python 3.8 ou superior: Download Python
    • Git: Download Git

    Instalação Passo a Passo

    1. Clonar o Repositório

    Abra o terminal e execute o seguinte comando para clonar o repositório do 2Do:

    git clone [URL_DO_REPOSITORIO]

    2. Instalar Dependências

    Navegue até o diretório do projeto e instale as dependências necessárias:

    cd 2Do
pip install -r requirements.txt

    Explicação do Arquivo config.ini

    O arquivo config.ini é o arquivo de configuração principal do 2Do. Ele permite personalizar aspectos do jogo, como resolução, volume de áudio, plano de fundo e controles. O arquivo é dividido em seções, cada uma contendo parâmetros específicos.

    Estrutura do Arquivo

    O arquivo config.ini segue o formato padrão de arquivos INI, com seções e pares de chave-valor. Abaixo está um exemplo do conteúdo do arquivo:

    [graphics]
resolution = 1280x720
fullscreen = no

[audio]
volume = 80

[background]
image = graphics/Background/Brown.png
x_block_bounds = 12,62
y_block_bounds = 10,37
scroll_speed = 0.6

    Seções e Parâmetros

    Seção [graphics]

    • resolution: Define a resolução da tela do jogo. O valor deve ser no formato LARGURAxALTURA (por exemplo, 1280x720).
    • fullscreen: Define se o jogo deve ser executado em tela cheia. Valores possíveis:
    • yes: Ativa o modo tela cheia.
    • no: Executa o jogo em modo janela.

    Seção [audio]

    • volume: Define o volume do áudio do jogo. O valor deve ser um número entre 0 (mudo) e 100 (volume máximo).

    Seção [background]

    • image: Define o caminho da imagem de fundo. O caminho deve ser relativo ao diretório do projeto.
    • x_block_bounds: Define os limites horizontais do plano de fundo. O valor deve ser no formato MIN,MAX (por exemplo, 12,62).
    • y_block_bounds: Define os limites verticais do plano de fundo. O valor deve ser no formato MIN,MAX (por exemplo, 10,37).
    • scroll_speed: Define a velocidade de rolagem do plano de fundo. Valores maiores resultam em rolagem mais rápida.

    Como Configurar Resolução, Volume e Plano de Fundo

    1. Configurando a Resolução

    Para alterar a resolução do jogo, edite o parâmetro resolution na seção [graphics]. Por exemplo, para definir a resolução para 1920x1080:

    [graphics]
resolution = 1920x1080

    2. Configurando o Volume

    Para ajustar o volume do jogo, edite o parâmetro volume na seção [audio]. Por exemplo, para definir o volume para 50%:

    [audio]
volume = 50

    3. Configurando o Plano de Fundo

    O plano de fundo no 2Do é configurado através da seção [background] no arquivo config.ini. Aqui estão os parâmetros que você pode ajustar:

    image

    Define o caminho da imagem de fundo. O caminho deve ser relativo ao diretório do projeto. Por exemplo:

    image = graphics/Background/NewBackground.png

    x_block_bounds e y_block_bounds

    Esses parâmetros definem os limites horizontais e verticais do plano de fundo. Eles controlam quantas vezes a imagem de fundo será repetida (ou "tileada") ao longo dos eixos X (horizontal) e Y (vertical).

    • x_block_bounds: Define os limites horizontais. O valor deve ser no formato MIN,MAX. Por exemplo, 12,62 significa que a imagem de fundo será repetida horizontalmente entre os blocos 12 e 62 do mapa.
    • y_block_bounds: Define os limites verticais. O valor deve ser no formato MIN,MAX. Por exemplo, 10,37 significa que a imagem de fundo será repetida verticalmente entre os blocos 10 e 37 do mapa.
    O que são "blocos"?

    No contexto do 2Do, um bloco é uma unidade de medida usada para posicionar elementos no jogo. Cada bloco tem um tamanho fixo, que é definido pelo tamanho dos tiles no mapa (por exemplo, 16x16 pixels). Os limites horizontais e verticais (x_block_bounds e y_block_bounds) determinam quantos blocos serão preenchidos com a imagem de fundo.

    Exemplo Prático

    Imagine que você tem uma imagem de fundo de 256x256 pixels e o tamanho de cada bloco no jogo é 16x16 pixels. Se você definir:

    x_block_bounds = 12,62
y_block_bounds = 10,37

    Isso significa que: - No eixo X (horizontal), a imagem de fundo será repetida 50 vezes (62 - 12 = 50 blocos). - No eixo Y (vertical), a imagem de fundo será repetida 27 vezes (37 - 10 = 27 blocos).

    Esses valores garantem que o fundo cubra toda a área visível do jogo, criando um efeito contínuo e sem bordas visíveis.

    scroll_speed

    Define a velocidade de rolagem do plano de fundo. Valores maiores resultam em rolagem mais rápida, enquanto valores menores criam um efeito mais suave. Por exemplo:

    scroll_speed = 0.6

    Exemplo Completo de Configuração de Plano de Fundo

    Aqui está um exemplo completo de como configurar o plano de fundo:

    [background]
image = graphics/Background/CustomBackground.png
x_block_bounds = 12,62
y_block_bounds = 10,37
scroll_speed = 0.5

    Neste exemplo: - A imagem CustomBackground.png será usada como plano de fundo. - O fundo será repetido horizontalmente entre os blocos 12 e 62. - O fundo será repetido verticalmente entre os blocos 10 e 37. - A velocidade de rolagem será de 0.5, criando um efeito de parallax suave.

    Dicas de Configuração

    • Teste diferentes valores para x_block_bounds e y_block_bounds: Ajuste esses valores para garantir que o fundo cubra toda a área do jogo sem cortes ou repetições indesejadas.
    • Use imagens de fundo compatíveis: Certifique-se de que as imagens de fundo tenham dimensões adequadas para evitar distorções ou bordas visíveis.
    • Ajuste a velocidade de rolagem: Valores menores de scroll_speed criam um efeito de parallax mais suave, enquanto valores maiores aumentam a sensação de movimento.
